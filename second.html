Welcome to your comprehensive guide to becoming a Python Full Stack Developer. This page will walk you through everything from the absolute basics of Python to building complex web applications with Django and other modern technologies. Let's get started!

Python Basics

Python is a powerful, readable, and versatile programming language. It's the foundation of our stack.

Variables & Data Types

Variables store data. Python has several built-in data types.

# String (text)
name = "Alice"

# Integer (whole number)
age = 30

# Float (decimal number)
height = 5.9

# Boolean (True/False)
is_student = True

# Printing variables
print(f"{name} is {age} years old.")

Control Flow (if/else)

Control flow directs the order in which code is executed.

if age >= 18:
    print("You are an adult.")
else:
    print("You are a minor.")

Functions

Functions are reusable blocks of code.

def greet(person_name):
    return f"Hello, {person_name}!"

message = greet("Bob")
print(message) # Output: Hello, Bob!

Collections (Lists & Dictionaries)

Collections store multiple items.

# List (ordered, changeable)
fruits = ["apple", "banana", "cherry"]
print(fruits[1]) # Output: banana

# Dictionary (unordered, key-value pairs)
person = {
    "name": "Charles",
    "age": 25
}
print(person["name"]) # Output: Charles

Frontend Basics

The frontend is what the user sees and interacts with in their browser. HTML, CSS, and JavaScript are the three core technologies.

HTML (HyperText Markup Language)

HTML provides the structure and content of a webpage.

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>
    <h1>Welcome!</h1>
    <p>This is a paragraph.</p>
    <a href="https://google.com">Go to Google</a>
</body>
</html>

CSS (Cascading Style Sheets)

CSS is used to style the HTML content, controlling colors, fonts, layout, and more.

/* This code would go in a .css file or a <style> tag */
body {
    font-family: Arial, sans-serif;
    background-color: #f0f0f0;
}

h1 {
    color: navy;
    text-align: center;
}

JavaScript

JavaScript makes webpages interactive. It can manipulate HTML and CSS, handle events, and much more.

// This code would go in a .js file or a <script> tag
document.addEventListener('DOMContentLoaded', () => {
    const heading = document.querySelector('h1');
    heading.addEventListener('click', () => {
        alert('You clicked the heading!');
    });
});

Frontend Frameworks

Frameworks provide pre-written code and tools to build UIs faster.

Bootstrap

Bootstrap is a popular CSS framework for building responsive, mobile-first websites. You add it via a CDN link and use its pre-defined classes.

<!-- Add Bootstrap CSS to your <head> -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- Use Bootstrap classes in your HTML -->
<div class="container mt-4">
    <h1 class="text-primary">Hello, Bootstrap!</h1>
    <button class="btn btn-success">Click Me</button>
</div>

Tailwind CSS

Tailwind is a utility-first CSS framework. Instead of pre-built components, it gives you low-level utility classes to build completely custom designs.

<!-- Setup is more involved, usually with a build step. -->
<!-- But in your HTML, you'd write something like this: -->

<div class="bg-slate-800 text-white p-8 rounded-lg shadow-lg">
    <h1 class="text-3xl font-bold mb-4">Hello, Tailwind!</h1>
    <button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
      Click Me
    </button>
</div>

The Django Framework

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It follows the "Don't Repeat Yourself" (DRY) principle.

Introduction & Setup

First, you need to set up a virtual environment and install Django.

# 1. Create a virtual environment
python -m venv myenv

# 2. Activate it
# On Windows:
myenv\Scripts\activate
# On MacOS/Linux:
source myenv/bin/activate

# 3. Install Django
pip install django

Start a Project & App

A Django **project** is the entire web application. An **app** is a module within the project that does one specific thing (e.g., a blog, a poll).

# 1. Start a new project named 'myproject'
django-admin startproject myproject

# 2. Navigate into the project directory
cd myproject

# 3. Start a new app named 'myapp'
python manage.py startapp myapp
After creating an app, you must add it to the `INSTALLED_APPS` list in your project's `settings.py` file.

Views & URLs

A **view** is a Python function that takes a web request and returns a web response. **URLs** map a web address to a specific view.

In `myapp/views.py`:

from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello, Django World!")

Create `myapp/urls.py`:

from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]

In `myproject/urls.py`, include your app's URLs:

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')), # Include myapp's URLs
]

Models & Database

A **model** is the single, definitive source of information about your data. It's a Python class that maps to a database table.

In `myapp/models.py`:

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title

After defining models, you need to create and apply database migrations:

# Create migration files
python manage.py makemigrations

# Apply migrations to the database
python manage.py migrate

The Django Admin

Django comes with a powerful, built-in admin interface. To use it, you need to register your models and create a superuser.

In `myapp/admin.py`:

from django.contrib import admin
from .models import Post

# Register your models here.
admin.site.register(Post)

Create a superuser to access the admin panel:

python manage.py createsuperuser

Now run the server (`python manage.py runserver`) and go to `/admin` to log in.

Static Files (CSS, JS, Images)

Static files are assets that don't change, like your CSS and JavaScript files. You need to configure Django to serve them.

In `myproject/settings.py`:

import os

STATIC_URL = '/static/'

# Add this line to tell Django where to find your static files
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

Create a `static` folder in your project's root directory. Inside it, you can place your CSS, JS, etc. Then, in your templates:

{% load static %}
<link rel="stylesheet" href="{% static 'css/style.css' %}">

Media Files (User-Uploaded Content)

Media files are user-uploaded files, like profile pictures or documents.

In `myproject/settings.py`:

import os

MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

In `myproject/urls.py`, add the URL pattern for development:

from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # ... your other urls
]

if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

Connect to MySQL

By default, Django uses SQLite. To switch to MySQL, first install a database driver.

pip install mysqlclient

Then, update the `DATABASES` setting in `myproject/settings.py`:

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'your_db_name',
        'USER': 'your_db_user',
        'PASSWORD': 'your_db_password',
        'HOST': 'localhost',  # Or your DB host
        'PORT': '3306',
    }
}

User Authentication

Django has a built-in authentication system for handling user accounts, groups, permissions, and cookie-based user sessions.

You don't need to create a User model; Django provides one. The `login`, `logout`, and `password_change` views are also built-in.

A simple login view in `myapp/views.py` might look like this:

from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login, logout
from django.contrib.auth.forms import AuthenticationForm

def login_view(request):
    if request.method == 'POST':
        form = AuthenticationForm(request, data=request.POST)
        if form.is_valid():
            username = form.cleaned_data.get('username')
            password = form.cleaned_data.get('password')
            user = authenticate(username=username, password=password)
            if user is not None:
                login(request, user)
                return redirect('home')
    else:
        form = AuthenticationForm()
    return render(request, 'login.html', {'form': form})

def logout_view(request):
    logout(request)
    return redirect('home')

Form Processing

Django provides two main ways to handle HTML forms.

1. Using `forms.py` (The Django Way)

This is the recommended approach. It handles rendering, validation, and cleaning data.

Create `myapp/forms.py`:

from django import forms

class ContactForm(forms.Form):
    name = forms.CharField(max_length=100)
    email = forms.EmailField()
    message = forms.CharField(widget=forms.Textarea)

In `myapp/views.py`:

def contact_view(request):
    if request.method == 'POST':
        form = ContactForm(request.POST)
        if form.is_valid():
            # Process the data in form.cleaned_data
            name = form.cleaned_data['name']
            # ... do something with the data
            return redirect('success')
    else:
        form = ContactForm()
    return render(request, 'contact.html', {'form': form})

2. Using `request.POST` (Manual Processing)

You can access form data directly from the `request.POST` dictionary. This is simpler for very basic forms but lacks built-in validation.

In `myapp/views.py`:

def simple_form_view(request):
    if request.method == 'POST':
        name = request.POST.get('name')
        email = request.POST.get('email')
        # Manually validate and process data
        print(f"Name: {name}, Email: {email}")
        return redirect('success')
    return render(request, 'simple_form.html')

REST API

A REST API allows your application to communicate with other services (like a mobile app or a JavaScript frontend). The best way to build APIs in Django is with the **Django REST Framework (DRF)**.

First, install it:

pip install djangorestframework

A basic API requires a **Serializer** (to convert model instances to JSON) and a **ViewSet** (which handles the logic for CRUD operations).

Example of a `serializers.py` and `views.py` for our `Post` model:

# myapp/serializers.py
from rest_framework import serializers
from .models import Post

class PostSerializer(serializers.ModelSerializer):
    class Meta:
        model = Post
        fields = ['id', 'title', 'content', 'created_at']

# myapp/views.py
from rest_framework import viewsets
from .models import Post
from .serializers import PostSerializer

class PostViewSet(viewsets.ModelViewSet):
    queryset = Post.objects.all()
    serializer_class = PostSerializer

Flask Basics

Flask is a "microframework" for Python. It's much simpler and more lightweight than Django, making it great for smaller projects or for learning web concepts.

Intro & Hello World

A complete Flask app can be just a few lines of code.

Install Flask: pip install Flask

Create a file `app.py`:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, World!'

@app.route('/user/<username>')
def show_user_profile(username):
    # Variable passing from the URL
    return f'User: {username}'

if __name__ == '__main__':
    app.run(debug=True)

Template Rendering

Flask uses the Jinja2 templating engine, which is very similar to Django's.

Create a `templates` folder, and inside it, an `index.html` file:

<!DOCTYPE html>
<html>
<head>
    <title>Flask Template</title>
</head>
<body>
    <h1>Hello, {{ name_variable }}!</h1>
    <p>Here is your list:</p>
    <ul>
        {% for item in list_variable %}
            <li>{{ item }}</li>
        {% endfor %}
    </ul>
</body>
</html>

Update your `app.py` to render this template:

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    # Passing variables to the template
    my_name = "Alice"
    my_list = ["apple", "banana", "cherry"]
    return render_template('index.html', name_variable=my_name, list_variable=my_list)

if __name__ == '__main__':
    app.run(debug=True)
