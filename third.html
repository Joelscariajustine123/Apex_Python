1. Python Basics

Python is a versatile and beginner-friendly language. It's the foundation of our stack.

Variables and Data Types

# Strings, Integers, Floats, Booleans
name = "Alice"
age = 30
height = 5.8
is_student = False

print(f"{name} is {age} years old.")

Data Structures

# List (mutable)
fruits = ["apple", "banana", "cherry"]
fruits.append("orange")

# Tuple (immutable)
coordinates = (10.0, 20.0)

# Dictionary (key-value pairs)
person = {"name": "Bob", "age": 25}
print(person["name"])

Control Flow

# If-Else statements
if age > 18:
    print("Adult")
else:
    print("Minor")

# For loop
for fruit in fruits:
    print(fruit)

# While loop
count = 0
while count < 3:
    print(f"Count is {count}")
    count += 1

Functions

def greet(name):
    return f"Hello, {name}!"

message = greet("Charlie")
print(message)

2. Frontend Basics

The frontend is what users see and interact with. It's built with HTML, CSS, and JavaScript.

2.1 HTML (HyperText Markup Language)

HTML provides the structure of your webpage.

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>
    <h1>Welcome!</h1>
    <p>This is a paragraph.</p>
    <a href="https://google.com">Go to Google</a>
    <img src="image.jpg" alt="An image">
</body>
</html>

2.2 CSS (Cascading Style Sheets)

CSS is used to style and visually format your HTML.

/* style.css */
body {
    font-family: Arial, sans-serif;
    background-color: #f4f4f4;
}

h1 {
    color: #333;
    text-align: center;
}

p {
    color: #666;
}

2.3 JavaScript

JavaScript makes your webpages interactive.

DOM Manipulation

// script.js
// Get an element and change its content

heading.textContent = 'Hello, World!';

// Add an event listener to a button
const myButton = document.getElementById('my-btn');
myButton.addEventListener('click', () => {

});

3. Django Framework

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It follows the Model-View-Template (MVT) architectural pattern.

3.1 Start Project & App

Let's create our Django project and a dedicated app inside it.

Commands

# Install Django
pip install django

# Create a new project named 'myproject'
django-admin startproject myproject
cd myproject

# Create a new app named 'myapp'
python manage.py startapp myapp

# Run the development server
python manage.py runserver

After creating an app, you must add it to the INSTALLED_APPS list in your project's settings.py file.

3.2 Views & URLs

Views handle the logic for requests, and URLs map request paths to those views.

myapp/views.py

from django.shortcuts import render
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello, Django World!")
    
def profile(request):
    context = {'name': 'Alice', 'age': 30}
    return render(request, 'myapp/profile.html', context)

myproject/urls.py

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')), # Include app's urls
]

myapp/urls.py (Create this file)

from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
    path('profile/', views.profile, name='profile'),
]

myapp/templates/myapp/profile.html

3.3 Models

Models are Python classes that define the structure of your database tables.

myapp/models.py

from django.db import models

class Product(models.Model):
    name = models.CharField(max_length=200)
    description = models.TextField(blank=True)
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name

Commands to apply model changes

# Create migration files for model changes
python manage.py makemigrations

# Apply migrations to the database
python manage.py migrate

3.4 Admin Panel

Django comes with a powerful, built-in admin interface. Register your models to manage them.

myapp/admin.py

from django.contrib import admin
from .models import Product

You need to create a superuser to access the admin panel. Run python manage.py createsuperuser and follow the prompts.

3.7 User Authentication

Django has a robust built-in authentication system.

Login View Logic (myapp/views.py)

from django.contrib.auth import authenticate, login, logout
from django.shortcuts import render, redirect

def login_view(request):
    if request.method == 'POST':
        # Logic to handle form submission
        username = request.POST.get('username')
        password = request.POST.get('password')
        user = authenticate(request, username=username, password=password)
        if user is not None:
            login(request, user)
            return redirect('home')
        else:
            # Handle invalid login
            pass
    return render(request, 'myapp/login.html')

def logout_view(request):
    logout(request)
    return redirect('home')

3.8 Form Processing

Django provides two main ways to handle forms: using the Form class or handling POST data directly.

Method 1: Using forms.py (Recommended)

myapp/forms.py (Create this file)
from django import forms

class ContactForm(forms.Form):
    name = forms.CharField(max_length=100)
    email = forms.EmailField()
    message = forms.CharField(widget=forms.Textarea)
myapp/views.py
from .forms import ContactForm

def contact_view(request):
    if request.method == 'POST':
        form = ContactForm(request.POST)
        if form.is_valid():
            # Process the data in form.cleaned_data
            name = form.cleaned_data['name']
            # ... do something with the data
            return redirect('success_page')
    else:
        form = ContactForm()
    return render(request, 'myapp/contact.html', {'form': form})
myapp/templates/myapp/contact.html
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">Submit</button>
</form>

Method 2: Handling request.POST directly

myapp/views.py
def simple_form_view(request):
    if request.method == 'POST':
        name = request.POST.get('name')
        email = request.POST.get('email')
        # Process the data...
        print(f"Name: {name}, Email: {email}")
        return redirect('success_page')
    return render(request, 'myapp/simple_form.html')

3.9 REST API with DRF

Django REST Framework (DRF) is the standard for building APIs with Django.

Installation

pip install djangorestframework

myapp/serializers.py (Create this file)

from rest_framework import serializers
from .models import Product

class ProductSerializer(serializers.ModelSerializer):
    class Meta:
        model = Product
        fields = ['id', 'name', 'price', 'description']

myapp/views.py

from rest_framework import viewsets
from .models import Product
from .serializers import ProductSerializer

class ProductViewSet(viewsets.ModelViewSet):
    queryset = Product.objects.all()
    serializer_class = ProductSerializer

myapp/urls.py

from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import ProductViewSet

router = DefaultRouter()
router.register(r'products', ProductViewSet)

urlpatterns = [
    # ... your existing urls
    path('api/', include(router.urls)),
]

4. Flask Basics

Flask is a micro web framework for Python. It's lightweight and flexible.

Installation

pip install Flask

app.py (A simple Flask app)

from flask import Flask, render_template

app = Flask(__name__)

# Basic route
@app.route('/')
def hello_world():
    return 'Hello, Flask!'

# Route with variable passing
@app.route('/user/<username>')
def show_user_profile(username):
    return f'User: {username}'
    
# Template rendering
@app.route('/profile')
def profile():
    user_data = {'name': 'Chris', 'is_admin': True}
    return render_template('profile.html', user=user_data)
    
if __name__ == '__main__':
    app.run(debug=True)

templates/profile.html

<!DOCTYPE html>
<html>
<head>
    <title>User Profile</title>
</head>
<body>
    <h1>Hello, {{ user.name }}!</h1>
    {% if user.is_admin %}
        <p>You have admin privileges.</p>
    {% else %}
        <p>You are a regular user.</p>
    {% endif %}
</body>
</html>
