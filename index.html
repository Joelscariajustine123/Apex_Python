Phase 1.1: Python Basics üêç

Before diving into Django, a strong foundation in Python is essential. Django is a Python framework, so all your backend logic will be written in Python.

Key Concepts to Master:

  • Data Types: Integers, Floats, Strings, Booleans.
  • Data Structures: Lists, Dictionaries, Tuples, Sets.
  • Control Flow: If-Elif-Else statements, For and While loops.
  • Functions: Defining functions (`def`), arguments, and return values.
  • Object-Oriented Programming (OOP): Classes, objects, inheritance, and methods.
  • Virtual Environments: Crucial for managing project dependencies.

Creating and Activating a Virtual Environment

# 1. Create a virtual environment named 'venv'
python -m venv venv

# 2. Activate it (Windows)
.\venv\Scripts\activate

# 3. Activate it (macOS/Linux)
source venv/bin/activate

# 4. Install packages (e.g., Django)
pip install django

Phase 1.2: Frontend Basics üé®

A full stack developer needs to be comfortable with the frontend. Django templates will generate HTML, which is styled by CSS and made interactive with JavaScript.

  • HTML5: The structure of your webpages. Understand semantic tags like `<header>`, `<nav>`, `<main>`, `<section>`, and `<footer>`.
  • CSS3: The styling. Learn about the box model, Flexbox, and Grid for creating modern layouts.
  • JavaScript (ES6+): For client-side interactivity. Focus on DOM manipulation, events, and making API calls (`fetch`).

Phase 1.3: Developer Tools üõ†Ô∏è

Professional developers use tools to manage their code and workflow efficiently.

  • Git & GitHub: Version control is non-negotiable. Learn how to `init`, `add`, `commit`, `push`, and `pull`.
  • Code Editor: VS Code is the most popular choice, with powerful extensions for Python and Django development.

Basic Git Workflow

# Initialize a new git repository in your project folder

# Commit your changes with a message
git commit -m "Initial project setup"

# Push your code to a remote repository (like GitHub)
git push origin main

Phase 2.1: Setup & First Project üöÄ

Let's get Django up and running. Once you've activated your virtual environment and installed Django (`pip install django`), you can create your first project and app.

Creating a Django Project and App

# 1. Create a new project called 'myproject'
django-admin startproject myproject
cd myproject

# 2. Create a new app called 'myapp'
python manage.py startapp myapp

# 3. Run the development server
python manage.py runserver

After running the server, visit `http://127.0.0.1:8000/` in your browser to see the Django welcome page.

Phase 2.2: The MVT Architecture

Django follows the Model-View-Template (MVT) architectural pattern, which is a variation of the classic Model-View-Controller (MVC).

  • Model: The data layer. Defines the structure of your data and handles interactions with the database (Create, Read, Update, Delete).
  • View: The business logic layer. It receives a web request, interacts with the model to fetch data, and passes that data to a template.
  • Template: The presentation layer. This is what the user sees. It's an HTML file with special Django syntax to display the data provided by the view.

Phase 2.3: Models & Database üì¶

Models are Python classes that map to your database tables. Django's Object-Relational Mapper (ORM) allows you to interact with your database using Python code instead of SQL.

myapp/models.py

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title

After creating or changing a model, you must create and apply migrations.

Applying Model Changes

# Create the migration files based on model changes
python manage.py makemigrations

# Apply the migrations to the database
python manage.py migrate

Phase 2.4: Views & URLs ‚öôÔ∏è

Views process requests and return responses. URLs map a web address to a specific view function.

myapp/views.py

from django.shortcuts import render
from .models import Post

def post_list(request):
    posts = Post.objects.all().order_by('-created_at')
    context = {'posts': posts}
    return render(request, 'myapp/post_list.html', context)

myproject/urls.py (Project URLs)

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')), # Include app URLs
]

myapp/urls.py (App URLs)

from django.urls import path
from . import views

urlpatterns = [
    path('', views.post_list, name='post_list'),
]

Phase 2.5: Templates (DTL) üìÑ

Templates are HTML files that use the Django Template Language (DTL) to display dynamic data.

You need to create a `templates` folder inside your app directory (`myapp/templates/myapp/`).

myapp/templates/myapp/post_list.html

<!DOCTYPE html>
<html>
<head>
    <title>My Blog</title>
</head>
<body>
    <h1>Latest Posts</h1>
    <hr>
    {% for post in posts %}
        <article>
            <h2>{{ post.title }}</h2>
            <p>{{ post.content|truncatewords:30 }}</p>
            <small>Published on: {{ post.created_at|date:"F j, Y" }}</small>
        </article>
        <hr>
    {% empty %}
        <p>No posts yet.</p>
    {% endfor %}
</body>
</html>

Phase 2.6: The Django Admin ‚≠ê

Django comes with a powerful, production-ready admin interface. You can easily manage your app's data by registering your models.

myapp/admin.py

from django.contrib import admin
from .models import Post

admin.site.register(Post)

You first need to create a superuser to log in.

Creating an Admin User

python manage.py createsuperuser

Now run the server and go to `/admin` to log in and manage your Posts.

Phase 3.1: Forms & User Input

from django import forms

class PostForm(forms.ModelForm):
    class Meta:

Phase 3.2: Static & Media Files

Static files are your CSS, JavaScript, and images. Media files are user-uploaded content.

You need to configure your `settings.py` to tell Django where to find these files.

myproject/settings.py (at the bottom)

import os

STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

Phase 3.3: User Authentication

Django includes a built-in authentication system for handling user accounts, groups, permissions, and cookie-based sessions.

  • Sign Up: Creating new user accounts.
  • Login/Logout: Authenticating users and managing sessions.
  • Password Management: Securely handling password changes and resets.

Phase 3.4: Class-Based Views (CBVs)

While function-based views are simple, Class-Based Views (CBVs) help you reuse code and structure your views in a more organized, object-oriented way. Django provides generic CBVs for common tasks.

myapp/views.py (CBV Example)

from django.views.generic import ListView, DetailView
from .models import Post

class PostListView(ListView):
    model = Post
    template_name = 'myapp/post_list.html'
    context_object_name = 'posts'

class PostDetailView(DetailView):
    model = Post
    template_name = 'myapp/post_detail.html'

Phase 4.1: REST APIs with DRF üåê

To build modern, decoupled applications (e.g., with a React or mobile frontend), you need to create a REST API. Django REST Framework (DRF) is the go-to library for this.

Install DRF

pip install djangorestframework

Key DRF Concepts:

  • Serializers: Convert complex data types, like Django models, into native Python datatypes that can then be easily rendered into JSON.
  • ViewSets & Routers: Work together to automatically generate the URL conf for your API endpoints.

Phase 4.2: Testing Your App üß™

Writing tests is crucial for building reliable and maintainable applications. Django has a built-in test runner that uses Python's standard `unittest` module.

myapp/tests.py

from django.test import TestCase
from django.urls import reverse
from .models import Post

class PostModelTest(TestCase):
    def test_post_creation(self):
        post = Post.objects.create(title="A Test Title", content="Some test content.")
        self.assertEqual(str(post), "A Test Title")

class PostListViewTest(TestCase):
    def test_view_url_exists_at_desired_location(self):
        response = self.client.get('/')
        self.assertEqual(response.status_code, 200)

Phase 4.3: Deployment ‚òÅÔ∏è

Taking your application from your local machine to a live web server is the final step. Popular hosting options include Heroku, DigitalOcean, AWS, and PythonAnywhere.

General Steps:

  • Web Server: Use a production-ready server like Gunicorn or uWSGI.
  • Database: Switch from the default SQLite to a more robust database like PostgreSQL.
  • Static Files: Configure a service like Amazon S3 or WhiteNoise to serve your static files.
  • Environment Variables: Never hard-code sensitive information (like secret keys or database passwords) in your code. Use environment variables.

Appendix: Django Commands Cheatsheet üìñ

A reference for the most common `python manage.py` commands you will use throughout your development process.

Project & App Management

  • startproject <project_name>: Creates a new Django project directory. (Note: this one uses `django-admin`, not `manage.py`).
  • startapp <app_name>: Creates a new Django app directory within your project.

Development Server

  • runserver: Starts the lightweight development web server on port 8000.
  • runserver 8080: Starts the server on a different port.
  • runserver 0.0.0.0:8000: Makes the server accessible on your local network.

Database Migrations

  • makemigrations: Scans your `models.py` files for changes and creates new migration files.
  • migrate: Applies all unapplied migrations to the database, updating the schema.
  • showmigrations: Lists all migrations in the project and their status.
  • sqlmigrate <app_name> <migration_number>: Shows the SQL statements for a specific migration.

User Management

  • createsuperuser: Creates an admin user with full access to the Django admin site.
  • changepassword <username>: Allows you to change a user's password.

Testing & Debugging

  • test: Runs the automated tests for all installed apps.
  • test <app_name>: Runs tests for a specific app.
  • shell: Opens the Python interactive shell with your Django project's environment loaded. Perfect for testing models and queries.
  • check: Checks your entire Django project for common problems.

Static Files

  • collectstatic: Gathers all static files from your apps into a single directory (`STATIC_ROOT`) for deployment.
